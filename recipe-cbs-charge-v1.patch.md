# Recipe v1 Patch — 应用 6 个补丁（基于 9B dry-run review）

> **触发**: 2026-06-30 9B dry-run 命中红线
> **应用时机**: 修完即覆盖 `recipe-cbs-charge-v1.md` → 重新跑 mock dry-run

---

## Patch 1: 增加最终报告前的证据反查规则（Evidence Backcheck）

**在生成最终报告前，必须执行 Evidence Backcheck**：

- 报告中的每个告警 ID 必须来自 R1-R5 中间结果
- 报告中的每个网元对象必须来自 R1-R5 中间结果
- 报告中的每个时间点必须来自 R1-R5 中间结果
- 报告中的每个错误码必须来自 R1-R5 中间结果
- 报告中的每个百分比必须来自 R1-R5 中间结果
- 报告中的每个 KPI 数值必须来自 R1-R5 中间结果
- **不允许把错误码编号误写成百分比**（例：5004 → 500%）
- **不允许把百分比、次数、对象名混写**

若任一字段无法反查，必须将该字段替换为"未在现有证据中确认"，**不得继续输出为事实**。

## Patch 2: 增加数字一致性校验

最终报告输出前必须检查：

- **错误码 ↔ 百分比** 必须保持一致（5004 → 87.0%；5005 → 9.6%；5009 → 3.4%）
- 错误码的**对象集中度**（1085/1085 表示 100% 在该对象）**不等于总失败笔数**
- 错误码的**总失败笔数**（1247）必须明确数据来源
- 成功率当前值必须在全文一致，**不得同时出现 96.3%、96.5% 等多个值**，除非分别标明时间点
- 所有数字必须保留**小数点对齐**（如 99.1、99.2、98.5、96.5、96.3 都是 1 位小数）

## Patch 3: 增加只读边界约束

**P0 阶段只允许输出只读诊断建议。**

**禁止输出**：
- ❌ 未声明工具命令
- ❌ `kubectl` / SQL / shell / ssh 等直接操作命令
- ❌ 重启 / 扩容 / 切流 / 降级等直接执行式动作
- ❌ 自动处置建议
- ❌ 未经人工确认的配置修改建议

**允许输出**：
- ✅ 建议人工确认的检查项
- ✅ 建议通过**受控运维工具**查看的指标
- ✅ 建议补充采集的数据
- ✅ 建议验证的观察项
- ✅ 风险提示

**处置建议模板**（替换原模板）：

```
### 处置建议（只读诊断版 — 需人工确认）
- 人工确认项：
  - 检查 OCS-BJ-02 连接池实时使用率
  - 检查 OCS-BJ-02 最大连接数配置
  - 检查 OCS-BJ-02 等待队列长度
  - 检查 OCS-BJ-02 连接失败日志
- 风险提示：
  - 在补充上述证据前，不要直接执行重启 / 扩容操作
```

## Patch 4: 调整推荐根因表达

**将"推荐根因"改为"推荐高置信候选根因"**。

**推荐格式**：

```
### 推荐高置信候选根因
- **候选 #N**：<简短描述>
- 置信度：高 / 中 / 低
- 已有支持证据：<R5 收集的关键事实>
- 仍缺失的直接证据：<列出>
- 需要补充验证：<列出>
- 在补充验证前，不输出最终确定根因
```

**禁止表达**（避免误以为确定根因）：
- ❌ "这是根因"
- ❌ "该根因已确认"
- ❌ "故障原因是 X"（不带"候选"或"疑似"前缀）

## Patch 5: 增加失败处理规则

**若最终报告校验失败，模型不得输出"校验通过"。**

**必须输出**：

```
## 校验失败报告
- 校验失败项：<V1-V5 哪一项>
- 失败字段：<具体字段>
- 失败原因：<具体原因>
- 建议修正：<对应 patch>
- 是否暂停进入下一阶段：是 / 否
```

## Patch 6: 增加自检不可作为唯一依据（确定性后置校验器）

**V1-V5 自检只能作为模型自评，不作为最终验收**。

**最终验收必须由外部 deterministic verifier 完成**，至少包括：

- ✅ 字段反查（每个字段是否来自 R1-R5 中间结果）
- ✅ 数值一致性（百分比、对象集中度、总笔数对齐）
- ✅ 工具白名单（只用了 7 个声明工具）
- ✅ 只读边界（无 kubectl/SQL/shell/重启/扩容）
- ✅ 报告结构完整性（8 节齐全）

**自检判定规则**：

- 自检 PASS 但确定性校验 FAIL → **以确定性校验为准**，必须修正后重出报告
- 自检 FAIL → 必须按 Patch 5 输出失败报告
- 自检 PASS + 确定性校验 PASS → 才算验收通过
